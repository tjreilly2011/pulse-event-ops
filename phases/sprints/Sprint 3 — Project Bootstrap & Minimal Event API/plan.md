**Layer contract:**
- `api/` → calls `application/` only
- `application/` → calls `infrastructure/` only
- `infrastructure/` → calls `domain/` structs only
- `domain/` → no external calls; pure data types and enums

### Backend (Rust)

#### API Endpoints

| Method | Path            | Handler             | Description             |
|--------|-----------------|---------------------|-------------------------|
| GET    | `/health`       | `health::health`    | Liveness check          |
| POST   | `/events`       | `events::create`    | Create new event        |
| GET    | `/events`       | `events::list`      | List all events         |
| GET    | `/events/:id`   | `events::get_by_id` | Fetch single event      |

#### Domain Models

**EventStatus** (stored as TEXT, serialised as SCREAMING_SNAKE_CASE)
```
CREATED | DELIVERED | ACKNOWLEDGED | IN_PROGRESS | RESOLVED | CANCELLED
```
All new events are inserted with `status = CREATED`. No other status transitions in this sprint.

**Event** (core fields — from Sprint 2 `event-payload.md`)
```
id                      UUID        generated by platform
event_type              TEXT        required — e.g. "passenger_assistance"
status                  TEXT        always CREATED on insert
created_by              UUID        required — actor ID (auth deferred)
created_at              TIMESTAMPTZ generated by platform
updated_at              TIMESTAMPTZ generated by platform
destination_location_id TEXT        required — routing target (TEXT; FK deferred)
source_location_id      TEXT        optional
title                   TEXT        optional
description             TEXT        optional
priority                TEXT        defaults to "normal"
vertical_metadata       JSONB       optional — stored and returned, not interpreted
```

#### Database Schema & Migration

File: `migrations/0001_create_events.sql`

- Enable `timescaledb` extension
- Create `events` table with `UNIQUE (id)` constraint (no simple UUID PK — required for TimescaleDB hypertable)
- Convert `events` to TimescaleDB hypertable on `created_at`
- Add index on `created_at DESC` and `id`

> TimescaleDB requires the partitioning column (`created_at`) to be part of or excluded from the primary key. Using `UNIQUE (id)` instead of `PRIMARY KEY (id)` satisfies this constraint while preserving UUID lookup uniqueness.

#### Crate Structure

Library + binary:
- `[lib]` exposes `create_app(pool: PgPool) -> Router` — enables integration testing without spin-up
- `[[bin]]` is a thin wrapper that loads config, connects DB, runs migrations, serves

## Key Technical Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Web framework | Axum 0.7 | Constitution-aligned; Tokio-native |
| DB access | sqlx 0.7 (dynamic queries) | No compile-time `DATABASE_URL` required at build; `query_as::<_, T>()` is typed via `FromRow` |
| Status storage | TEXT (not Postgres ENUM) | Extensible without migrations; sqlx::Type handles encode/decode |
| `created_by` | UUID | Clean for future auth integration; tests use a fixed UUID |
| `destination_location_id` | TEXT | No `locations` table yet; TEXT with future FK noted in migration comment |
| `vertical_metadata` | JSONB | Stored and returned as-is; platform does not interpret per constitution |
| TimescaleDB PK | `UNIQUE (id)` (not `PRIMARY KEY`) | Required to allow hypertable partitioning on `created_at` |
| Integration tests | `#[sqlx::test]` | Real DB, fresh per test, torn down automatically |
